# Yii 3

Yii 3 improves on Yii 2's container and encourages dependency injection but still uses ActiveRecord.

- **Better DI.** Use the built-in DI container to define ports and bind adapters in config files.
- **Still ActiveRecord-centric.** Follow the same rules as Yii 2: domain logic does not belong in AR models.
- **Less mature ecosystem.** Expect gaps in official packages. Build small boundaries before they become churn points.
